# S12RYT WebUI 初始方向

本倉庫目前用於沉澱 WebUI 的設計方向、元件風格與頁面規則，讓後續 AI 或開發者在新增頁面時有一致依據。

## 使用方式

1. 先閱讀本文件了解整體定位。
2. 依產品需求從根目錄的方案資料夾中挑選一個方向：
   - `方案-01-沉穩企業SaaS控制台`
   - `方案-02-暗色AI工作台`
   - `方案-03-清爽工具型WebUI`
   - `方案-04-品牌行銷Landing`
   - `方案-05-資料可視化監控`
3. 進入選定方案資料夾，依照該資料夾內的 `README.md`、`design-system.md`、`components.md`、`pages.md` 實作。
4. 若需求混合多種方向，請先指定主方案，再只借用其他方案的單一元素，避免風格漂移。

## 共通 WebUI 原則

- 以桌面與平板體驗為主，手機版至少確保可閱讀、可操作、無水平捲動。
- 色彩、字體、圓角、陰影、間距需集中定義，不要每個頁面臨時決定。
- 所有可互動元素需有 hover、focus、disabled 狀態。
- 不使用 emoji 當正式 UI icon；使用一致的 SVG icon 系統。
- 文字階層需明確：頁面標題、區塊標題、內文、輔助文字不能混用。
- 重要操作要有確認、載入中、成功、失敗狀態。
- 表格、卡片、表單、空狀態、錯誤狀態都要有可複用模式。

## 建議預設技術方向

若尚未決定框架，建議先用下列抽象規則描述 UI：

- Styling：Tailwind CSS 或同等 utility-first 設計 token。
- Icons：Lucide / Heroicons 擇一。
- Charts：Recharts / ECharts 擇一，依資料量決定。
- Components：按 `Button`、`Input`、`Card`、`Dialog`、`Tabs`、`Table`、`Toast`、`EmptyState` 建立基礎元件。

## 方案選擇速查

| 方案 | 適合場景 | 第一印象 | 主要風格 |
| --- | --- | --- | --- |
| 方案 01 | B2B 後台、管理系統、SaaS | 穩定、可信、專業 | 淺色、藍灰、資料密度中高 |
| 方案 02 | AI 工具、開發者平台、模型工作台 | 高科技、專注、沉浸 | 暗色、霓虹點綴、命令介面感 |
| 方案 03 | 小工具、設定面板、效率產品 | 清楚、快速、無負擔 | 極簡、白底、高留白 |
| 方案 04 | 官網、產品介紹、轉換頁 | 有品牌感、易理解 | 大視覺、漸層、敘事分區 |
| 方案 05 | 監控中心、分析平台、營運儀表板 | 即時、精準、掌控感 | 深淺皆可、圖表優先、狀態色明確 |

## AI 實作提示

每次讓 AI 新增 WebUI 時，請附上：

```text
請使用「方案-XX」作為唯一主視覺方向，遵守該資料夾內 design-system.md、components.md、pages.md 的規則。
不要混用其他方案的色彩、圓角、陰影與排版密度，除非我明確指定。
```
