# 項目表

## 專案名稱

`s12ryt-webui`

## 目前狀態

- 本地目錄一開始不是 git repository。
- GitHub 遠端倉庫目前為 empty repository。
- Issue #1 要求：在倉庫根目錄給出多種 WebUI 方案並用資料夾分類，讓 AI 後續知道 WebUI 方向與統一風格。

## 根目錄結構

| 路徑 | 用途 |
| --- | --- |
| `README.md` | 專案總入口，說明如何選擇方案與共通 WebUI 原則。 |
| `方案-01-沉穩企業SaaS控制台/` | B2B SaaS、管理後台、CRM、權限與訂閱管理等穩定專業方向。 |
| `方案-02-暗色AI工作台/` | AI chat、agent 控制台、prompt playground、模型測試與開發者工具方向。 |
| `方案-03-清爽工具型WebUI/` | 單頁工具、轉換器、設定器、產生器等輕量效率方向。 |
| `方案-04-品牌行銷Landing/` | 官網、產品介紹、等待名單、價格頁等轉換導向方向。 |
| `方案-05-資料可視化監控/` | 監控中心、營運儀表板、分析平台、告警與即時狀態方向。 |
| `agent/` | AI 接手紀錄與專案管理文件。 |

## 每個方案資料夾內容

| 檔案 | 用途 |
| --- | --- |
| `README.md` | 方案定位、適用/不適用場景、實作優先順序。 |
| `design-system.md` | 色彩、字體、間距、layout、互動與禁用規則。 |
| `components.md` | 元件規格與狀態規則。 |
| `pages.md` | 頁面結構範本與 AI 提示模板。 |

## 建議依賴

目前尚未建立實際 Web App，因此沒有 package manager 或 runtime 依賴。若後續要實作，建議：

- Framework：React / Next.js / Vue 其一。
- Styling：Tailwind CSS 或等價 design token 系統。
- Icons：Lucide 或 Heroicons 擇一。
- Charts：Recharts / ECharts 擇一。
- Components：先建立基礎 Button、Input、Card、Dialog、Tabs、Table、Toast、EmptyState。

## 維護規則

- 新增方案時必須建立同樣四份文件：`README.md`、`design-system.md`、`components.md`、`pages.md`。
- 修改某方案風格時，優先改該方案 `design-system.md`，不要只改頁面範例。
- 任何實際 UI 實作都要先指定主方案，避免混用造成風格不一致。
