## 建議依賴

目前尚未建立實際 Web App，因此沒有 package manager 或 runtime 依賴。若後續要實作，建議：

- Framework：React / Next.js / Vue 其一。
- Styling：Tailwind CSS 或等價 design token 系統。
- Icons：Lucide 或 Heroicons 擇一，不混用多套 icon。
- Charts：Recharts / ECharts 擇一。
- Components：先建立 Button、Input、Card、Dialog、Tabs、Table、Toast、EmptyState、Skeleton、Tooltip。
- Theme：將方案中的 CSS token 映射成程式碼中的 theme。

## 維護規則

- 新增方案時必須建立同樣四份文件：`README.md`、`design-system.md`、`components.md`、`pages.md`。
- 修改某方案風格時，優先改該方案 `design-system.md`，不要只改頁面範例。
- 任何實際 UI 實作都要先指定唯一主方案，避免混用造成風格不一致。
- 若借用其他方案元素，只能借用單一頁型或元件概念，不可混用主色、狀態色、圓角、陰影、資訊密度與大面積背景效果。
- 所有正式 UI icon 都使用 SVG icon system，不使用 emoji。
