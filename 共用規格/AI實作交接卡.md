# AI 實作交接卡

本文件提供每次把 WebUI 方案交給 AI 或開發者實作前的交接模板。它的目的不是取代各方案文件，而是把「選哪套方案、做哪個頁面、哪些邊界不能碰」一次講清楚。

## 使用方式

1. 先從 `方案選擇矩陣.md` 選定唯一主方案。
2. 再用本文件填好交接卡。
3. 將交接卡連同主方案資料夾、`AI輸出契約.md`、`細化檢查清單.md` 一起交給 AI。
4. 若要使用跨方案小組件，補上 `小組件/` 來源文件與元件名稱，但視覺 token 仍回到主方案。
5. 若 AI 輸出與交接卡衝突，以交接卡與主方案文件為準。

## 常見交接錯誤

### 只說「做一個漂亮後台」

問題：AI 不知道是 SaaS、監控、金融、電商後台，容易混用風格。

改成：

```text
主方案：方案-01-沉穩企業SaaS控制台
頁面：客戶列表頁
主要任務：搜尋、篩選、批次停用客戶
資料密度：中高
禁用風格：不要霓虹、不要大型 hero、不要遊戲化動效
```

### 同時要求多套方案

問題：會混主色、密度與互動語言。

改成：

```text
主方案：方案-07-電商營運購物體驗
允許借用元素：只借用方案-05 的營運 KPI 圖表概念，用於商家後台首頁
不可借用：方案-05 的深色監控背景與告警色密度
```

### 沒有狀態設計

問題：AI 只做 happy path，實際使用會破碎。

改成：

```text
必備狀態：
- loading：商品卡 skeleton
- empty：沒有商品時引導新增商品
- error：載入失敗可重試
- disabled：庫存不足不可結帳
- success：加入購物車 toast
- permission denied：無權編輯商品時顯示唯讀提示
```

## 快速複製版

```text
請使用「方案-XX」作為唯一主方案。
本次要做：
目標使用者：
主要任務：
資料密度：
必備內容：
必備狀態：loading、empty、error、disabled、success、permission denied。
RWD：桌面、平板、手機都需說明。
可借用元素：無 / 僅借用「方案-XX」的「某元件概念」。
小組件來源：無 / 使用 `小組件/某文件.md` 的某元件。
小組件套用方式：只取結構、狀態與互動規則，視覺 token 回到主方案。
禁用風格：不要 emoji icon、不要混用其他方案色彩/圓角/陰影/資訊密度。
請先輸出方案確認、資訊架構、Design token 使用、元件清單、狀態設計、RWD 策略、可及性檢查，最後才輸出實作內容。
```
