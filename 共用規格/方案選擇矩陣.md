# 方案選擇矩陣

本文件用來在實作前選定唯一主方案。先判斷產品核心任務，再判斷資訊密度、信任需求、視覺強度與互動節奏。

## 快速決策樹

1. 使用者主要在看數據、處理資料、管理資源嗎？
   - 是一般後台：選 `方案-01-沉穩企業SaaS控制台`。
   - 是即時監控、告警、圖表：選 `方案-05-資料可視化監控`。
   - 是金流、帳務、投資、支付：選 `方案-06-高信任金融科技平台`。
2. 使用者主要在和 AI、模型、prompt 或 agent 互動嗎？
   - 選 `方案-02-暗色AI工作台`。
3. 使用者主要完成單一工具任務嗎？
   - 選 `方案-03-清爽工具型WebUI`。
4. 使用者主要被說服、導流、留下聯絡方式或購買嗎？
   - 官網/產品介紹：選 `方案-04-品牌行銷Landing`。
   - 商品購買：選 `方案-07-電商營運購物體驗`。
   - 活動任務/遊戲化轉換：選 `方案-12-遊戲化活動互動介面`。
5. 使用者主要閱讀、搜尋、學習或參與內容嗎？
   - 文件/知識庫：選 `方案-09-知識庫文件中心`。
   - 課程/學習：選 `方案-11-教育學習課程平台`。
   - 社群/論壇：選 `方案-08-社群內容與論壇平台`。
6. 使用者主要欣賞作品、案例、媒體或個人品牌嗎？
   - 選 `方案-10-創作者作品集媒體展示`。

## 混用規則

允許借用：

- 元件概念，例如在 `方案-01` 借用 `方案-09` 的文章閱讀區塊。
- 單一頁面模式，例如在 `方案-07` 商品後台中借用 `方案-05` 的營運圖表。
- 文案結構，例如在 `方案-04` 中借用 `方案-10` 的案例敘事。

不允許混用：

- 主色與狀態色系統。
- 圓角尺度與陰影風格。
- 資訊密度節奏。
- 大面積背景效果，例如把 AI 工作台霓虹背景放進金融頁。
- 元件互動語言，例如在企業後台中使用遊戲化彈跳與獎勵動效。
