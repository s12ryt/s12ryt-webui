# 細化檢查清單

本清單用於檢查 WebUI 方案是否足夠細化。任何新頁面或新方案都應符合以下項目。

## 1. 方案定位

- [ ] 有明確產品類型與使用者角色。
- [ ] 有 3 個以上典型使用情境。
- [ ] 有「適合」與「不適合」場景。
- [ ] 有資訊密度定義：低、中、中高、高。
- [ ] 有情緒關鍵字，例如可信、沉浸、快速、活躍、精準。

## 2. Design token

- [ ] 背景色、表面色、邊框色、主文字、次文字皆有 token。
- [ ] primary、success、warning、danger、info 皆有 token。
- [ ] hover、active、focus、disabled、loading 狀態有定義。
- [ ] 圓角至少有 sm、md、lg。
- [ ] 陰影至少有 card、popover、modal 或明確禁止陰影。
- [ ] 字體有中文字體 fallback，不只寫英文字體。
- [ ] 數字、金額、代碼是否需要等寬字體有說明。

## 3. Layout

- [ ] 定義頁面最大寬度或說明不限制寬度的原因。
- [ ] 定義桌面、平板、手機的欄數變化。
- [ ] 定義 header、sidebar、content、footer 的尺寸與行為。
- [ ] 定義固定元素是否會遮擋內容。
- [ ] 定義空白區與資料密度，不讓卡片間距隨意漂移。

## 4. 元件

- [ ] Button 至少有 primary、secondary、ghost、danger。
- [ ] Form 有 label、helper、error、disabled、loading。
- [ ] Card 有標題、內容、action、空資料狀態。
- [ ] Table 或 List 有 loading、empty、error、pagination 或 infinite scroll。
- [ ] Dialog/Drawer 有確認、取消、關閉、危險操作規則。
- [ ] Toast/Notification 有成功、失敗、警告、資訊狀態。
- [ ] 每個方案至少有 3 個情境專用元件。

## 5. 頁面模板

- [ ] 至少定義 Dashboard / List / Detail / Settings 或該方案對應的核心頁。
- [ ] 每個頁面有資訊順序，不只列元件名稱。
- [ ] 有主要 CTA 與次要 CTA 位置規則。
- [ ] 有錯誤、空資料、權限不足、載入中情境。
- [ ] 有 AI 提示模板，能直接複製使用。

## 6. 內容與文案

- [ ] 文案語氣符合方案定位。
- [ ] 錯誤訊息有說明原因與下一步。
- [ ] 危險操作文案不含糊，例如「刪除此專案」而非「確定」。
- [ ] 金額、日期、狀態、百分比格式一致。
- [ ] 不使用 emoji 代替正式 icon 或狀態。

## 7. 互動與動效

- [ ] 動效有 duration 與 easing 範圍。
- [ ] hover 不造成 layout shift。
- [ ] focus ring 對鍵盤使用者清楚可見。
- [ ] 可取消或尊重 `prefers-reduced-motion`。
- [ ] 高風險操作需要二次確認或復原機制。

## 8. RWD 與可及性

- [ ] 375px 寬度無水平捲動。
- [ ] 768px 平板有合理欄位折疊。
- [ ] 1024px 以上桌面利用空間但不過度拉長文字。
- [ ] 文字對比至少符合 WCAG AA。
- [ ] 表單 input 有 label。
- [ ] 圖片有 alt，裝飾圖可空 alt。
- [ ] 不只用顏色表達狀態，需搭配文字或 icon。

## 9. 交付前檢查

- [ ] 明確說明使用哪一個主方案。
- [ ] 沒有混入其他方案的視覺語言。
- [ ] 產出包含 token、layout、components、states、page structure。
- [ ] 產出可以被下一個 AI 接手，不依賴口頭記憶。
