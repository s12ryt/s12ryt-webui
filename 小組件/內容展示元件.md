# 內容展示元件

這一組元件用於 Landing、作品集、知識庫、活動頁與內容導向頁面。它們負責敘事節奏、視覺層級與轉換引導。

## HeroSection

### 用途

建立第一印象、主訴求與主要 CTA。

### 規則

- 必須先說明產品是什麼、給誰用、能解決什麼。
- 只保留一個最重要的主要 CTA。
- 避免過度裝飾而削弱訴求。

## FeatureGrid

### 用途

整理功能、優勢、賣點或差異化。

### 規則

- 每個 feature 要有具體行為或價值，不可只寫形容詞。
- 一致的卡片節奏比大量不同卡片更重要。
- 若 feature 太多，應分組或分頁，不要一次塞滿。

## PricingCard

### 用途

呈現方案、方案比較、價格與升級誘因。

### 規則

- 必須區分主推方案與其他方案。
- 價格卡內的 CTA 要明確對應下一步。
- 不可只寫價格，卻沒有內容差異與限制說明。

## MediaCard

### 用途

展示圖片、影片、專案、案例、作品或封面。

### 規則

- 圖片要有 alt 或說明文字。
- 大圖內容不應缺乏文字支撐。
- 重要資訊不可只靠視覺說明。

## ArticleCard

### 用途

承載文章、文件、教學、FAQ、公告或更新紀錄。

### 規則

- 標題、摘要、作者、日期與標籤要有一致層級。
- 長文內容應搭配閱讀時間或章節導覽。
- 文件頁應支援搜尋、版本或更新狀態。

## Timeline

### 用途

描述流程、歷程、變更紀錄、任務進度、事件序列。

### 規則

- 時間點要按清楚順序排列。
- 每個節點要有動作或結果，而不是純時間戳。
- 重要狀態變化應可回溯。

## Callout / Highlight

### 用途

強調關鍵訊息、提醒、建議或限制。

### 規則

- Callout 不可過多，否則會失去重點。
- 用於重要補充，而不是替代主要內容。

## QuoteBlock / Testimonial

### 用途

用於社會證明、使用者回饋、品牌敘事或內容引用。

### 規則

- 必須清楚標示來源。
- 引用內容要真實可讀，不要過度包裝。

## 使用建議

- Landing：`HeroSection` + `FeatureGrid` + `PricingCard`
- 作品集：`MediaCard` + `QuoteBlock` + `Timeline`
- 文件中心：`ArticleCard` + `Callout`
- 活動頁：`HeroSection` + `FeatureGrid` + `Timeline`
