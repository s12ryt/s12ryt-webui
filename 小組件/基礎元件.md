# 基礎元件

基礎元件是所有方案都會用到的最小 UI 單位。這些元件只定義責任與狀態，不直接指定風格；實作時必須套用主方案的 token。

## Button

### 用途

承載明確操作，例如送出、儲存、取消、刪除、重新整理、前往下一步。

### 建議 variants

- `primary`：頁面主要 CTA，每個主要區塊最多一個。
- `secondary`：次要操作，例如取消、返回、查看詳情。
- `ghost`：低干擾操作，例如更多、篩選、展開。
- `danger`：刪除、停用、撤銷授權等高風險操作。
- `link`：文字型導向，不應取代主要 CTA。

### 必備狀態

- default
- hover
- focus
- active
- disabled
- loading
- danger confirming

### 規則

- Loading 時保留原本寬度，避免按鈕文字改變造成 layout shift。
- Danger 操作不得只寫「確定」，需寫明動作，例如「刪除專案」。
- 同一個視覺區塊不要出現多個 `primary`。
- icon button 必須有可讀 label 或 `aria-label`。

## TextInput

### 用途

輸入短文字、搜尋、名稱、URL、email、token 等。

### 必備組成

- label
- input
- placeholder
- helper text
- validation message
- optional prefix/suffix

### 必備狀態

- default
- focus
- filled
- disabled
- readonly
- error
- success
- loading 或 validating

### 規則

- Placeholder 不可取代 label。
- Error 要說明原因與下一步。
- 密碼、token、金鑰欄位需提供顯示/隱藏或複製規則。
- 搜尋輸入要有 clear 行為。

## Select

### 用途

從有限選項中選擇單一或多個值。

### 必備組成

- label
- trigger
- option list
- selected state
- empty option state
- helper/error text

### 規則

- 選項超過 8 個時應支援搜尋或分類。
- 多選需要清楚顯示已選數量。
- 不可只用顏色標示選中，需有文字、勾選或狀態符號。
- 鍵盤需能開啟、上下移動、選取、關閉。

## Checkbox / Radio / Switch

### 用途

- Checkbox：多選或同意條款。
- Radio：互斥選項。
- Switch：立即生效的二元設定。

### 規則

- Switch 不應用於需要送出表單才生效的設定。
- Checkbox label 必須可點擊。
- Radio group 必須有群組標題。
- 危險設定切換需有確認或復原。

## Card

### 用途

承載一組資訊或操作，是多數頁面最常見的內容容器。

### 必備組成

- header：標題、描述、狀態或 action。
- body：主要內容。
- footer：次要操作或補充資訊，可省略。

### 狀態

- default
- hoverable
- selected
- disabled
- loading
- empty
- error

### 規則

- Card 不一定都可點擊；可點擊時必須有清楚 hover/focus。
- 不要把太多不相關操作塞進同一張 Card。
- 高密度方案中 Card 間距較小，但仍需有清楚分隔。

## Badge / Tag / Pill

### 用途

標示狀態、分類、等級、風險、版本、標籤。

### 常見 variants

- neutral
- info
- success
- warning
- danger
- pending
- archived

### 規則

- 狀態 Badge 必須搭配文字，不只靠顏色。
- 不同狀態色需回到主方案 token。
- 不要把 Badge 當按鈕，除非明確設計為 filter chip。

## Avatar

### 用途

表示使用者、團隊、作者、商家、講師、客服等身份。

### 規則

- 無圖片時使用文字縮寫或預設圖形，不使用 emoji。
- 必須定義圖片載入失敗狀態。
- 群組 Avatar 超過 3-5 個時需折疊為 `+N`。

## Divider

### 用途

分隔內容群組，降低資訊混淆。

### 規則

- 不要用大量 Divider 補救混亂 layout；應先調整資訊架構。
- 垂直 Divider 在手機上通常要改為間距或水平分隔。

## Tooltip

### 用途

補充短說明，不承載關鍵資訊。

### 規則

- Tooltip 內容應短於 2 行。
- 重要錯誤、表單規則、法律資訊不可只放 Tooltip。
- 必須支援鍵盤 focus 觸發。
