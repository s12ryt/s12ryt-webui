# 導覽與浮層元件

這一組元件負責頁面之間的路徑、區域切換與臨時浮層互動。它們會影響資訊架構與鍵盤操作，因此要特別注意可及性與遮罩行為。

## AppHeader

### 用途

放置品牌、全站搜尋、通知、使用者入口與全域操作。

### 規則

- 不要把所有動作都塞進 header。
- 桌面與手機的 header 應有明確折疊規則。
- 若有全站搜尋，需說明作用範圍與快捷鍵。

## SideNav

### 用途

管理多頁產品的主要導覽、分類與權限可見性。

### 規則

- 桌面可固定，平板可收合，手機通常轉 drawer。
- active、expanded、disabled 狀態要明確。
- 分組層級不要過深，否則要考慮 breadcrumbs 或 secondary nav。

## Breadcrumbs

### 用途

顯示層級、返回路徑與目前位置。

### 規則

- 最多保留可讀層級，不要塞太多階層。
- 行動版可縮短顯示，但要保留返回能力。
- 不可只作裝飾，必須真的能導回上層。

## TabsNav

### 用途

在同一頁切換區段、內容分類或設定分頁。

### 規則

- tab 數量太多時應改成 side nav、segmented control 或 filter bar。
- active tab 要有清楚焦點與 keyboard 支援。
- 每個 tab 的內容量應接近，避免切換後節奏差異太大。

## Dialog

### 用途

用於確認、編輯、查看細節、警告或高風險操作。

### 規則

- 必須 trap focus，Esc 或關閉按鈕要可用。
- 標題、說明、主要操作、次要操作要清楚。
- 危險操作要有 confirm text 或二次確認。

## Drawer

### 用途

適合側邊編輯、詳情檢視、篩選器與次要流程。

### 規則

- Drawer 不應遮蔽所有操作的上下文。
- 手機上常可取代 dialog；桌面上可與主內容並存。
- 關閉時要維持原本頁面位置。

## PopoverMenu

### 用途

承載少量上下文動作，例如更多、列操作、使用者選單。

### 規則

- 選項過多時應改成 drawer 或 dialog。
- 點選外部、Esc 與鍵盤導覽要可預期。
- 危險動作不要和一般動作混在一起而無區隔。

## Tooltip

### 用途

提供短說明、縮寫解釋、欄位補充或圖示補註。

### 規則

- Tooltip 不可承載關鍵資訊。
- 文字需短，不適合放長段落。
- 必須支援 hover 與 focus；手機上需有替代方案。

## 使用順序建議

- 多頁後台：`AppHeader` + `SideNav` + `Breadcrumbs`
- 設定頁：`TabsNav` + `Dialog` + `Drawer`
- 資料列操作：`PopoverMenu` + `Tooltip`
