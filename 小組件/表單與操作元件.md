# 表單與操作元件

這一組元件負責輸入、篩選、批次操作、流程確認與提交行為。它們是管理後台、設定頁、建立流程與工具型 UI 的核心。

## FormField

### 用途

統一 label、input、helper、error、description 的包裝方式。

### 規則

- label 不能只放 placeholder。
- 錯誤訊息要說明原因與修正方法。
- 必填與選填要有一致標示。

## FieldGroup / FieldSet

### 用途

整理一組相關欄位，例如地址、付款資訊、帳號設定。

### 規則

- 群組標題要說明這組欄位的任務。
- 不要把互不相關的欄位塞在同一群組。
- 複雜表單要分段，避免長頁失去方向。

## SearchInput

### 用途

搜尋、即時篩選、關鍵字查找。

### 規則

- 清除按鈕要明確。
- 若搜尋會觸發高成本查詢，要有 debounce 或明確提交動作。
- 必須支援鍵盤輸入與快捷鍵提示（若有）。

## FilterBar

### 用途

承載狀態篩選、日期區間、分類、排序與標籤條件。

### 規則

- 篩選數量太多時要有收合策略。
- 顯示已套用條件與清除全部操作。
- 需要區分瞬時篩選與需送出後才生效的篩選。

## ActionBar / BulkActionBar

### 用途

對已選資料做批次操作，例如刪除、匯出、標記、轉移。

### 規則

- 必須顯示已選數量。
- 危險批次操作要比單筆操作更明確。
- 選擇過多時需要顯示取消選取與重設。

## Pagination

### 用途

資料分頁、頁數跳轉、每頁筆數切換。

### 規則

- 要清楚說明目前範圍，例如 21-40 / 312。
- 若採 infinite scroll，需說明何時使用與回到上一段的方式。
- 不要讓分頁控制過小、難點擊。

## Stepper

### 用途

用於多步驟流程，例如結帳、註冊、匯入、設定精靈。

### 規則

- 必須標示每一步的目的，而不只是步驟名稱。
- 可回退的步驟要有清楚規則。
- 不要用過多步驟讓流程失去完成感。

## Confirm pattern

### 用途

高風險提交、刪除、停用、覆蓋、送出前確認。

### 規則

- 必須說明結果、影響範圍與是否可復原。
- 不要只顯示「確定/取消」，要具體寫動作。
- 危險操作可搭配輸入確認或延遲處理。

## 常見搭配

- 表格頁：`SearchInput` + `FilterBar` + `BulkActionBar` + `Pagination`
- 設定頁：`FormField` + `FieldGroup` + `Confirm pattern`
- 流程頁：`Stepper` + `FormField` + `ActionBar`
