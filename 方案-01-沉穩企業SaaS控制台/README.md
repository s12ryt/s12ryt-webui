# 方案 01：沉穩企業 SaaS 控制台

## 定位

適合 B2B SaaS、管理後台、CRM、ERP、權限管理、訂閱管理、內部營運系統、客服後台、審核平台等需要高可信度、高效率與高可讀性的產品。

## 設計關鍵字

- 專業
- 穩定
- 清楚
- 可預期
- 高效率
- 可維護
- 權限明確

## 視覺摘要

以淺色介面為主，搭配藍色作為主要行動色、灰階作為資訊骨架。整體不追求炫技，而是讓使用者快速理解資訊、完成操作、降低出錯。資料表格、篩選器、狀態標籤與批次操作是本方案的核心。

## 典型使用者

- 每天長時間操作後台的營運、客服、業務或管理者。
- 需要管理帳號、權限、訂閱、帳務、審核流程的內部人員。
- 需要查看 KPI、追蹤任務、處理例外狀況的主管。

## 適合頁面

- 儀表板首頁
- 使用者 / 團隊管理
- 權限與角色設定
- 訂閱與帳務
- 客戶 / CRM 列表
- 稽核紀錄
- 通知中心
- 審核工作台
- 系統設定

## 不適合

- 強品牌情緒的行銷頁。
- 遊戲、娛樂、潮流風產品。
- 需要大量沉浸式視覺效果的 AI 展示頁。
- 以大圖與故事為主的作品集頁面。

## 資訊密度

中高。桌面可以同時顯示 KPI、表格、篩選、狀態與操作；手機版應保留查看與基本操作，複雜批次操作可降級或移到桌面。

## 實作優先順序

1. 先建立全域 layout：側邊欄、頂部列、內容容器、breadcrumb。
2. 建立資料操作元件：表格、篩選器、分頁、狀態標籤、批次操作列。
3. 建立表單與流程元件：Dialog、Drawer、Stepper、Toast、Confirm。
4. 建立例外情境：empty、loading、error、permission denied、audit log。
5. 補儀表板與報表：KPI、趨勢圖、近期活動、待辦提醒。

## 風格邊界

- 可以有輕陰影與柔和卡片，但不做厚重玻璃擬態。
- 可以用狀態色，但表格內不能到處紅黃綠造成疲勞。
- 可以做 dashboard，但圖表不是主角，主要任務仍是管理與操作。
