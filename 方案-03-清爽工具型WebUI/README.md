# 方案 03：清爽工具型 WebUI

## 定位

適合線上小工具、轉換器、設定器、內容產生器、單一任務型 Web App、內部效率工具、查詢器、格式化工具與輕量設定頁。核心目標是讓使用者在最短時間理解輸入、完成操作、取得結果，並且不被多餘裝飾或複雜流程干擾。

## 設計關鍵字

- 極簡
- 直接
- 乾淨
- 快速
- 輕量
- 低學習成本
- 可複製結果
- 可重複操作

## 視覺摘要

大量留白、低裝飾、清楚輸入與結果區。使用者一進來就知道要輸入什麼、按哪裡、結果在哪裡。畫面應像一個可信任的精準工具，而不是大型產品官網或複雜後台。

## 典型使用者

- 想快速完成單一任務的一般使用者。
- 需要重複執行轉換、查詢、產生內容的工作者。
- 內部團隊成員，需要低成本完成格式化、驗證、計算、設定等工作。
- 開發者或營運人員，需要貼上資料並快速得到可複製輸出。

## 適合頁面

- Converter / Generator
- 設定表單
- 單頁工具
- 文件格式化 / 解析
- 小型查詢工具
- 檔案轉換工具
- 文字處理工具
- 計算器
- API 測試輕量工具
- 內部設定產生器

## 不適合

- 大型多角色後台。
- 高品牌敘事 Landing。
- 需要炫酷沉浸感的產品展示。
- 即時告警與高密度監控中心。
- 需要長時間社群互動的內容平台。

## 資訊密度

低到中。預設只顯示主流程必需資訊，進階設定、範例、歷史紀錄、說明文件應可收合或放在次要區塊。

## 使用情境範例

- JSON formatter：輸入 JSON，格式化、驗證並複製。
- Prompt generator：填入欄位，產生可複製 prompt。
- Image metadata checker：上傳圖片後顯示尺寸與格式。
- UTM builder：填入 campaign 參數後產生網址。
- Config builder：選擇選項後輸出 YAML / JSON。

## 實作優先順序

1. 建立單頁 layout：標題、說明、輸入區、操作列、輸出區。
2. 補齊輸入驗證、錯誤提示、loading、disabled、empty state。
3. 加入 copy、download、reset、load example 等高頻工具操作。
4. 加入進階設定，但預設收合，不干擾主流程。
5. 視需要補歷史紀錄、範例與 FAQ。

## 風格邊界

- 可以有清爽插圖或簡單圖示，但不可把視覺焦點從工具本身移開。
- 不使用厚重側邊欄與多層導覽。
- 不做複雜儀表板式 KPI 牆。
- 不用 emoji 當正式 UI icon。
