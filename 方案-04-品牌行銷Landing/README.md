# 方案 04：品牌行銷 Landing

## 定位

適合產品官網、服務介紹、等待名單、功能展示、價格頁、下載頁、Launch page、活動宣傳頁與品牌轉換頁。目標是讓訪客快速理解產品價值、建立信任、降低疑慮，最後完成註冊、預約、購買、下載或加入等待名單。

## 設計關鍵字

- 品牌感
- 信任
- 敘事
- 轉換
- 記憶點
- 情緒張力
- 清楚 CTA
- 社會證明

## 視覺摘要

使用強烈 hero、清楚 CTA、漸層或品牌圖形建立第一印象。頁面區塊要像故事：痛點、解法、功能、證明、價格、FAQ、CTA。每個區塊只回答一個轉換問題，不堆砌過多產品細節。

## 典型使用者

- 第一次接觸產品的潛在客戶。
- 正在比較方案或價格的決策者。
- 需要被說服留下 email 或試用的訪客。
- 從廣告、社群、搜尋結果進站的冷流量。
- 既有使用者想了解新功能或版本發布。

## 適合頁面

- 首頁 Landing
- Feature page
- Pricing page
- Changelog / Launch page
- Waitlist page
- Download page
- Campaign page
- Webinar / event registration
- Case study list
- Product announcement

## 不適合

- 需要長時間操作的工作台。
- 資料密度高的管理後台。
- 純工具單頁。
- 監控中心或告警系統。
- 長篇文件與 API reference。

## 資訊密度

低。每一屏只保留一個核心訊息與一個主要 CTA。詳細資訊可透過 FAQ、comparison table、modal 或子頁承接。

## 轉換目標範例

- 申請試用。
- 加入等待名單。
- 預約 Demo。
- 下載 App。
- 購買方案。
- 聯絡業務。
- 閱讀案例研究。

## 實作優先順序

1. 定義 hero 的目標對象、價值主張與主 CTA。
2. 建立區塊節奏：問題、解法、功能、信任、價格、CTA。
3. 補社會證明、FAQ、風險降低文案與 footer。
4. 補 mobile conversion path，確保 CTA 在小螢幕可見。
5. 優化載入速度、圖片尺寸、SEO 與可及性。

## 風格邊界

- 可以有強烈視覺、漸層、產品 mockup，但不可犧牲可讀性。
- 不要每個區塊都使用不同風格。
- 不要把 Landing 做成後台 dashboard。
- 不用 emoji 當正式 icon。
- 文案不可空泛，必須具體說明對象、場景、成果。
