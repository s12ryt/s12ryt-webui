# Components：品牌行銷 Landing

## Navbar

組成：

- 左側品牌名 / logo。
- 中間主要導覽 3-5 項。
- 右側 CTA。
- 手機版 menu button 與 drawer。

狀態：

- transparent over hero
- scrolled / solid background
- active anchor
- mobile drawer open

規則：

- Navbar 不應遮住 hero 重點文字。
- CTA 文案需與頁面主要轉換一致。
- 手機 menu item 必須容易點擊。

## Hero

必備：

1. Eyebrow：一句分類或亮點。
2. 強主標題：直接說出價值。
3. 副標：補充對象、情境、成果。
4. Primary CTA + Secondary CTA。
5. 視覺展示：產品截圖、mockup、流程圖或動態預覽。
6. Trust hint：客戶數、低風險說明、logo wall 入口。

狀態：

- CTA loading
- email submit success / error
- video preview open

## CTA Button

類型：

- Primary：開始試用、加入等待名單、預約 Demo。
- Secondary：觀看 Demo、查看價格、閱讀文件。
- Text link：深入了解、查看案例。

規則：

- 全頁主要 CTA 文案保持一致。
- Primary CTA 不要在同一區塊出現多個競爭選項。
- 表單型 CTA 必須有隱私或低風險說明。

## Feature Card

- 一張卡只講一個利益點。
- icon 使用一致 SVG icon。
- 標題要具體，不要抽象。
- 內文 1-2 句即可。
- 可附 small proof，例如「平均節省 40% 時間」。

## Benefit Section

用途：把功能翻譯成使用者成果。

結構：

1. Section title。
2. Pain / before。
3. Benefit / after。
4. Feature support。
5. Visual proof。

## Product Screenshot / Demo Frame

規則：

- 使用一致 browser frame 或 device frame。
- 圖片需有固定尺寸避免 layout shift。
- 截圖上的文字要可讀。
- 若使用影片，需提供 poster 與不自動播放聲音。

## Social Proof

可使用：

- 客戶 logo 牆。
- 使用者評語。
- 數據指標。
- 安全 / 合規標章。
- Case study 摘要。

規則：

- Logo 需灰階或一致色彩處理。
- Testimonial 要有姓名、角色或可驗證上下文。
- 數據指標要說明統計範圍。

## Pricing Card

必備：

- 方案名稱。
- 適合對象。
- 價格與計費週期。
- 主要功能。
- 限制與額度。
- CTA。

規則：

- 推薦方案可使用 primary border 或 badge。
- 不要隱藏關鍵限制。
- Enterprise / contact sales 需清楚說明適用情境。

## Comparison Table

- 用於 Pricing 或替代方案比較。
- 欄位不要過多，手機可改 accordion。
- 差異點需具體，不只寫「更多功能」。

## FAQ

- 使用 accordion。
- 問題需貼近轉換前疑慮。
- 答案短而具體。
- 常見疑慮：價格、取消、資料安全、導入時間、支援、限制。

## Waitlist / Lead Form

欄位：

- email
- name 或 company 可選
- use case 可選

狀態：

- idle
- submitting
- success
- invalid email
- duplicate signup
- server error

規則：

- 表單不宜過長。
- 送出後明確說明下一步。
- 隱私文案要靠近表單。

## Final CTA

- 重申核心價值。
- 一個主要 CTA。
- 可附低風險說明，例如「不需信用卡」。
- 背景可比其他區塊更強，但不可降低文字對比。

## Footer

包含：

- 產品 / 公司資訊。
- 主要連結。
- 法務連結。
- 聯絡方式。
- 社群或文件入口。
