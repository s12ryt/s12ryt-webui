# Pages：品牌行銷 Landing

## Home Landing

用途：產品或服務的主轉換頁。

建議區塊順序：

1. Navbar。
2. Hero：價值主張、CTA、產品視覺。
3. Logo wall / proof：快速建立信任。
4. Problem：明確描述痛點。
5. Solution：說明產品如何解決。
6. Feature grid：3-6 個核心利益。
7. Product screenshot：展示實際畫面。
8. How it works：三步驟流程。
9. Testimonials / case study。
10. Pricing teaser 或 plan cards。
11. FAQ。
12. Final CTA。
13. Footer。

必要狀態：

- CTA loading
- form success / error
- mobile menu
- FAQ open

## Feature Page

用途：針對單一功能或使用情境深入說明。

結構：

1. Feature hero：單一功能價值。
2. Use cases：3 個使用情境。
3. How it works：三步驟。
4. 深入截圖展示。
5. Benefit metrics。
6. Related features。
7. CTA。

設計重點：

- 每個 feature 都要對應明確成果。
- 不要只列功能名稱，要說明何時使用。

## Pricing Page

用途：幫助使用者選方案並降低購買疑慮。

結構：

1. Pricing hero。
2. Billing toggle。
3. 2-4 張 pricing cards。
4. Feature comparison table。
5. FAQ。
6. Security / trust note。
7. Contact sales CTA。

必要狀態：

- monthly / yearly toggle
- selected plan
- checkout loading
- contact form success / error

## Waitlist Page

用途：產品未正式推出前收集名單。

結構：

1. 強 hero。
2. Email form。
3. 三個核心亮點。
4. 預期上線時間或 roadmap。
5. Social proof 或 early access 說明。
6. Privacy note。

設計重點：

- 表單欄位越少越好。
- 成功送出後說明下一步。
- 不要承諾不確定的上線日期。

## Launch / Changelog Page

用途：新功能發布、版本更新、產品公告。

結構：

1. Launch hero：版本或功能亮點。
2. What is new：主要更新。
3. Before / after。
4. Screenshots / demo。
5. Migration or setup notes。
6. CTA：開始使用或閱讀文件。

## Case Study Page

用途：用真實故事建立信任。

結構：

1. Customer hero：客戶、產業、成果。
2. Challenge。
3. Solution。
4. Implementation details。
5. Results metrics。
6. Quote。
7. Related CTA。

## Campaign Page

用途：廣告、活動或特定客群轉換。

結構：

1. Audience-specific hero。
2. Offer or promise。
3. Proof。
4. Short feature list。
5. Form or CTA。
6. FAQ / objection handling。
