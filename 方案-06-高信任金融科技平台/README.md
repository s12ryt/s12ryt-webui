# 方案 06：高信任金融科技平台

## 定位

適合 FinTech、錢包、支付、投資、帳務、訂閱收款、交易紀錄、風險控管、企業財務面板等需要高度信任感與精準資訊的產品。

## 設計關鍵字

- 安全
- 精準
- 克制
- 可追溯
- 高信任
- 低噪音

## 視覺摘要

以深藍、墨綠或中性灰作為信任底色，搭配少量金色或亮藍作為重點行動色。整體避免浮誇炫技，優先讓金額、狀態、風險提示與操作確認清楚可辨識。

## 典型使用者

- 需要查看收支、餘額、投資績效的一般使用者。
- 需要核對交易、匯出報表、審核付款的財務人員。
- 需要監控風險、異常交易、權限操作的管理者。

## 適合頁面

- 帳戶總覽與資產配置
- 交易紀錄列表
- 付款/轉帳流程
- 發票與帳務中心
- 投資組合與收益分析
- 風險提示與審核工作台
- KYC / KYB 身份驗證流程

## 不適合

- 遊戲化、娛樂化過重的產品。
- 需要大量潮流視覺或強烈品牌情緒的 landing。
- 不需處理金額、交易、風險或敏感資料的一般工具。

## 資訊密度

中高。桌面可同時呈現餘額、趨勢、交易、風險提示；手機版要優先保留金額、狀態與下一步操作，其他資訊折疊。

## 實作優先順序

1. 先建立安全可信的 layout：頂部帳戶切換、內容容器、風險提示區。
2. 建立金額與交易元件：BalanceCard、TransactionList、AmountInput、StatusBadge。
3. 建立高風險互動：二次確認、OTP/驗證碼、審核流程、操作紀錄。
4. 最後補圖表與報表：資產配置、收益曲線、月度收支。

## 風格邊界

- 可以有細緻陰影與淡色提示，不做霓虹光暈。
- 可以用綠色表示收益或安全，但不能只靠顏色表示漲跌。
- 可以使用圖表，但不讓圖表搶過主要金額與風險提示。
