# Components：高信任金融科技平台

## Button

- Primary：主要付款、確認、送出審核。每個流程頁最多一個。
- Secondary：返回、儲存草稿、下載報表。
- Ghost：交易列內操作、查看詳情。
- Danger：刪除付款方式、撤銷授權、凍結帳戶。

尺寸：

- sm：高度 34px，左右 padding 12px。
- md：高度 42px，左右 padding 18px。
- lg：高度 50px，左右 padding 22px，用於付款確認。

規則：

- loading 時顯示 spinner 與文字，例如「付款處理中」。
- 高風險 primary 需搭配確認摘要，不可直接執行。
- danger button 需有明確動詞，不用「確定」。

## Balance Card

用於顯示帳戶餘額、可用餘額、凍結金額或投資總值。

結構：

1. 帳戶名稱與幣別。
2. 主金額。
3. 與上期相比或可用/凍結拆分。
4. 主要操作：轉入、轉出、付款。
5. 安全狀態或最後更新時間。

狀態：

- loading：金額 skeleton，不顯示假數字。
- hidden：可隱藏金額，用遮罩與「顯示」按鈕。
- restricted：顯示帳戶受限 banner。

## Transaction List / Table

欄位：

- 交易名稱 / 對象
- 類型
- 時間
- 金額
- 手續費
- 狀態
- 操作

規則：

- 金額右對齊，使用 tabular nums。
- 正負值需有符號與文字，不只靠顏色。
- 狀態使用 Badge：成功、處理中、待審核、失敗、已退款。
- 點擊列可開啟交易明細 drawer。
- 批次操作需清楚顯示選取筆數與總金額。

## Amount Input

- 支援幣別 prefix。
- 自動格式化千分位，但不要干擾輸入游標。
- 顯示可用餘額、最低/最高限制、預估手續費。
- 錯誤狀態需說明原因，例如「超過單筆轉帳上限 NT$ 100,000」。

## Risk Banner

用於風險提示、KYC 未完成、異常登入、匯率變動。

層級：

- info：一般提醒。
- warning：需要注意但可繼續。
- danger：阻擋操作，需要先處理。

內容必須包含：標題、原因、下一步、可選 CTA。

## Confirmation Panel

高風險操作前的確認摘要。

必備資訊：

1. 操作類型。
2. 收款方或對象。
3. 金額與幣別。
4. 手續費與匯率。
5. 預計完成時間。
6. 是否可取消。
7. 驗證方式。

## Verification Code / OTP

- 6 位數輸入框，可貼上完整代碼。
- 顯示倒數與重新發送。
- 錯誤次數過多要顯示鎖定時間。
- 不在錯誤訊息中洩漏敏感資訊。

## Status Badge

- Completed：綠色，文字「已完成」。
- Pending：藍色或灰色，文字「處理中」。
- Review：橘色，文字「待審核」。
- Failed：紅色，文字「失敗」。
- Refunded：灰色，文字「已退款」。

## Chart Card

適合：資產配置、收益曲線、月收支。

規則：

- 圖表下方需有數值摘要。
- 漲跌需同時顯示符號、文字與顏色。
- tooltip 顯示時間、數值、幣別。
- 不使用過多炫彩漸層。

## Empty State

- 沒有交易：提供「新增付款方式」或「建立第一筆交易」。
- 沒有報表：提供「調整日期範圍」。
- 尚未驗證：提供「開始身份驗證」。

## Toast / Inline Alert

- 金融流程錯誤優先 inline alert，toast 只作輔助。
- 成功 toast 應包含可追蹤資訊，例如交易編號或查看明細。
- 失敗訊息要給下一步，不只寫「操作失敗」。
