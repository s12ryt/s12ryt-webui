# 方案 07：電商營運購物體驗

## 定位

適合電商前台、商品列表、購物車、結帳流程、訂單查詢、促銷活動、商家營運後台與商品管理平台。

## 設計關鍵字

- 清楚
- 轉換
- 商品導向
- 可比較
- 快速下單
- 營運彈性

## 視覺摘要

以商品內容為主角，介面不搶商品圖片與價格資訊。色彩可比企業後台更有活力，但仍需維持購買流程可信與可預期。CTA 要清楚，價格、庫存、優惠、配送與退貨資訊不能被弱化。

## 典型使用者

- 正在瀏覽、比較、收藏與下單的消費者。
- 管理商品、庫存、訂單與促銷的商家。
- 分析銷售、轉換率、客單價的營運人員。

## 適合頁面

- 商品列表與分類頁
- 商品詳情頁
- 購物車與結帳流程
- 訂單查詢與退換貨
- 商品管理後台
- 促銷活動管理
- 營運銷售儀表板

## 不適合

- 嚴肅金融交易或法遵審核頁。
- 純文件閱讀型知識庫。
- 大量即時監控告警中心。

## 資訊密度

中。商品列表需可快速掃描，詳情頁需清楚分區，結帳頁需降低干擾；營運後台可提高密度但仍保留商品縮圖與狀態。

## 實作優先順序

1. 建立商品卡與價格呈現規則。
2. 建立列表篩選、排序、分頁或 infinite scroll。
3. 建立購物車與結帳流程的可信互動。
4. 補營運後台：訂單、商品、庫存、促銷、銷售指標。

## 風格邊界

- 可使用促銷色，但不能讓全頁都像廣告。
- 可使用大圖與卡片 hover，但不造成版面跳動。
- 前台偏轉換，後台偏效率；兩者共享品牌色但元件密度不同。
