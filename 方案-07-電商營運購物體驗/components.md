# Components：電商營運購物體驗

## Button

- Primary：加入購物車、立即購買、提交訂單。
- Secondary：加入收藏、查看更多、繼續購物。
- Ghost：商品卡快捷操作、表格列操作。
- Danger：取消訂單、刪除商品、移除優惠。

規則：

- 商品卡內 primary 可全寬。
- 結帳頁 primary 固定描述下一步，例如「前往付款」而非「下一步」。
- 缺貨時 disabled 文案要明確。

## Product Card

結構：

1. 商品圖片。
2. 標籤：新品、熱銷、折扣、低庫存。
3. 商品名稱。
4. 價格與原價。
5. 評分或銷量。
6. 快捷 CTA。

狀態：

- default：一般商品。
- sale：顯示折扣與原價。
- out-of-stock：圖片降低飽和，CTA 改「通知我」。
- selected：比較或批次操作時顯示勾選狀態。

## Product Gallery

- 支援主圖、縮圖、放大檢視。
- 圖片比例固定。
- 手機可左右滑動。
- Alt 需描述商品，不寫「圖片」。

## Price Block

必備：

- 現價。
- 原價與折扣（若有）。
- 稅金/運費提示。
- 會員價或優惠條件。

規則：

- 價格使用 tabular nums。
- 折扣文字不應比現價更搶眼。
- 免費不能只寫 0，需寫「免費」。

## Filter Sidebar / Drawer

篩選項：

- 分類
- 價格區間
- 品牌
- 規格
- 評分
- 庫存
- 促銷

規則：

- 桌面左側固定，手機 drawer。
- 已套用篩選需顯示 chip，可單獨移除。
- 空結果提供清除篩選 CTA。

## Cart Drawer

結構：

1. 商品列表。
2. 數量調整。
3. 優惠碼。
4. 小計、運費、折扣、總計。
5. 結帳 CTA。

規則：

- 移除商品需可復原或二次確認。
- 數量改變需即時更新總價。
- 庫存不足 inline 顯示在該商品。

## Checkout Stepper

步驟：

1. 聯絡資訊。
2. 配送方式。
3. 付款方式。
4. 訂單確認。

規則：

- 使用者可返回上一階段。
- 已完成步驟顯示摘要。
- 錯誤步驟需標記並聚焦第一個錯誤欄位。

## Order Summary

- 商品小計。
- 優惠折扣。
- 運費。
- 稅金。
- 總金額。
- 預估到貨時間。

桌面可 sticky 在右側；手機可折疊，但總金額與付款 CTA 要可見。

## Admin Product Table

欄位：

- 商品縮圖
- 名稱 / SKU
- 分類
- 庫存
- 價格
- 狀態
- 更新時間
- 操作

規則：

- 庫存低於門檻顯示 warning badge。
- 批次操作需確認影響商品數。
- 編輯商品使用 drawer 或獨立 detail page。

## Promotion Badge

- Sale：折扣。
- New：新品。
- Limited：限量。
- Free Shipping：免運。

不要使用過多 badge，商品卡最多 2 個。

## Empty State

- 商品列表空：建議調整篩選。
- 購物車空：提供繼續購物 CTA。
- 訂單空：提供瀏覽商品或查看活動。

## Toast

- 加入購物車成功：顯示商品名與「查看購物車」。
- 優惠碼套用失敗：inline 顯示在優惠碼欄位，toast 可輔助。
- 付款失敗：顯示原因與重新嘗試，不只寫失敗。
