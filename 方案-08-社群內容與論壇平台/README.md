# 方案 08：社群內容與論壇平台

## 定位

適合社群平台、論壇、會員社群、內容 feed、討論串、留言系統、通知中心、創作者互動區與內部知識交流平台。

## 設計關鍵字

- 活躍
- 可讀
- 可參與
- 有脈絡
- 人味
- 持續互動

## 視覺摘要

以內容可讀性與互動入口為中心。介面需要讓使用者快速理解誰發了什麼、何時發、有哪些回應、自己可以做什麼。色彩可比企業後台活潑，但不可讓互動按鈕搶過內容本身。

## 典型使用者

- 發文、回覆、收藏、追蹤的社群成員。
- 維護版面秩序、審核內容的管理員。
- 經營內容、觀察互動成效的創作者或社群經理。

## 適合頁面

- 首頁 Feed
- 討論串詳情
- 發文編輯器
- 通知中心
- 私訊或簡易訊息頁
- 使用者個人頁
- 版主管理與檢舉審核

## 不適合

- 嚴肅金流交易頁。
- 單一工具型輸入輸出頁。
- 圖表密集的監控中心。

## 資訊密度

中。Feed 需要可快速掃描，但每則內容要保留作者、時間、社群脈絡與互動數；討論串可提高密度，個人頁可降低密度突出身份。

## 實作優先順序

1. 建立 FeedItem 與 ThreadView。
2. 建立 Composer、CommentBox、ReactionBar。
3. 建立通知、檢舉、審核與權限狀態。
4. 補個人頁、搜尋、主題分類與收藏。

## 風格邊界

- 可以活潑，但不能像遊戲活動頁一樣過度刺激。
- 互動按鈕要清楚，但不要讓讚數、分享數壓過內容。
- 社群內容需要節制分隔線與留白，避免閱讀疲勞。
