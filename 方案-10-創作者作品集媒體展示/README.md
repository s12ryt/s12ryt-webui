# 方案 10：創作者作品集媒體展示

## 定位

適合作品集、攝影集、設計師個人網站、案例展示、媒體品牌、創作者首頁、專案故事、影像內容展示與高視覺記憶點的展示型 WebUI。

## 設計關鍵字

- 展示
- 沉浸
- 品牌個性
- 大圖
- 節奏感
- 案例敘事

## 視覺摘要

以作品、圖片、影片與案例故事為主角。介面本身要像展示場：留白、比例、字體與過場節奏都為內容服務。適合更強烈的品牌風格，但仍要維持導覽清楚、文字可讀與載入體驗穩定。

## 典型使用者

- 想展示作品的設計師、攝影師、插畫家、開發者或工作室。
- 想瀏覽案例、評估合作可能的客戶。
- 想閱讀創作過程與媒體內容的觀眾。

## 適合頁面

- 作品集首頁
- Gallery / Grid
- Case Study 詳情
- About / Profile
- Services / Collaboration
- Media article
- Contact / Inquiry

## 不適合

- 高密度企業後台。
- 金融交易或嚴肅審核流程。
- 文件知識庫長文閱讀作為主體。

## 資訊密度

低中。首頁與作品列表偏低密度，案例詳情可中密度但要以章節敘事呈現，不要像後台一樣塞滿卡片。

## 實作優先順序

1. 建立視覺節奏：Hero、作品 grid、留白、字體。
2. 建立媒體元件：MediaCard、Gallery、Lightbox、CaseStudyBlock。
3. 建立轉換入口：Contact CTA、服務介紹、合作流程。
4. 補載入、圖片 fallback、SEO 與可及性。

## 風格邊界

- 可以大膽，但不能犧牲可讀性。
- 可以使用動效，但不可讓作品載入不穩或干擾閱讀。
- 不應把每個作品卡都加重陰影或過多裝飾。
