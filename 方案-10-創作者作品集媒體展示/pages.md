# Pages：創作者作品集媒體展示

## Portfolio Home

結構：

1. Media hero：一句強品牌定位與代表作品。
2. Featured work：3-6 個精選作品。
3. Services / capabilities。
4. Case study teaser。
5. About preview。
6. Testimonials。
7. Contact CTA。

規則：

- 首屏只放一個主要訊息。
- 精選作品要有明確排序，不要全部等權重。
- 聯絡 CTA 不可藏太深。

## Gallery Page

結構：

1. Page title。
2. Category filters。
3. Gallery grid / masonry。
4. Lightbox。
5. Pagination 或 lazy loading。

規則：

- 圖片比例預先定義。
- 篩選切換時保留視覺穩定。
- 空結果提供清除篩選。

## Case Study Detail

結構：

1. Case header：標題、摘要、角色、年份、成果。
2. Hero media。
3. Challenge。
4. Process。
5. Selected visuals。
6. Outcome / metrics。
7. Testimonial。
8. Next project。
9. Contact CTA。

規則：

- 不是只放圖片，要有敘事脈絡。
- 成果數字要有單位與上下文。
- 重要圖片需有 alt。

## About Page

結構：

1. Portrait / visual statement。
2. Bio。
3. Skills / services。
4. Experience timeline。
5. Clients / collaborations。
6. Values / working style。
7. Contact CTA。

## Services Page

結構：

1. 服務定位。
2. Service cards。
3. Process steps。
4. Pricing range 或合作方式。
5. FAQ。
6. Inquiry CTA。

## Contact Page

結構：

1. 簡短合作邀請。
2. Contact form。
3. Project type selector。
4. Budget/timeline optional。
5. Direct email / social links。
6. Response expectation。

規則：

- 表單欄位不要過多。
- 成功送出需說明下一步。
- 錯誤需 inline 顯示。

## Media Article Page

結構：

1. Article hero。
2. Metadata。
3. Article body。
4. Inline media。
5. Related articles。
