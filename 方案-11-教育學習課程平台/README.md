# 方案 11：教育學習課程平台

## 定位

適合線上課程、學習平台、企業訓練、教學系統、知識付費、測驗平台、學習進度管理與教練陪跑型產品。

## 設計關鍵字

- 有進度
- 有陪伴
- 可完成
- 清楚路徑
- 即時回饋
- 降低挫折

## 視覺摘要

以學習路徑與完成感為核心。畫面需要讓使用者知道目前在哪裡、下一步做什麼、已完成多少、遇到問題如何回到正軌。色彩可以友善溫暖，但不能幼稚或干擾閱讀。

## 典型使用者

- 學習課程、觀看影片、完成作業與測驗的學生。
- 建立課程、批改作業、查看進度的老師或教練。
- 管理企業訓練完成率與能力地圖的管理者。

## 適合頁面

- 課程首頁
- 課程詳情與大綱
- 學習播放器
- 課程章節列表
- 測驗與作業
- 學習進度儀表板
- 教師/管理後台

## 不適合

- 高壓金融交易流程。
- 強銷售 landing 作為唯一目的的頁面。
- 大量即時監控告警。

## 資訊密度

中。學習頁應避免過多干擾；管理後台可提高密度。課程卡與章節列表要清楚呈現進度與下一步。

## 實作優先順序

1. 建立課程卡、章節列表與進度元件。
2. 建立學習播放器與筆記/資源區。
3. 建立測驗、作業、回饋與完成狀態。
4. 補教師後台、學習分析與提醒。

## 風格邊界

- 可以用進度與成就感，但不要過度遊戲化。
- 可以有插圖，但不要使用 emoji 當正式 icon。
- 學習內容閱讀與影片觀看不可被促銷或過多動效干擾。
