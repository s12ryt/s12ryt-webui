# 方案 12：遊戲化活動互動介面

## 定位

適合活動頁、任務中心、簽到、抽獎、積分、徽章、排行榜、挑戰賽、成長任務、會員促活與短期互動轉換頁。

## 設計關鍵字

- 有趣
- 即時回饋
- 任務導向
- 獎勵感
- 活動節奏
- 明確下一步

## 視覺摘要

以任務、進度、獎勵與即時回饋為中心。視覺可以比其他方案更鮮明，但仍需維持操作清楚與規則透明。重點是讓使用者知道「現在可以做什麼、完成後得到什麼、還差多少」。

## 典型使用者

- 參加活動、完成任務、領獎勵的一般使用者。
- 設計任務、設定獎勵、查看成效的營運人員。
- 參與排行榜、挑戰或社群活動的會員。

## 適合頁面

- 活動首頁
- 任務中心
- 簽到頁
- 積分與獎勵商城
- 排行榜
- 抽獎 / 轉盤 / 開獎流程
- 活動營運後台

## 不適合

- 金融交易、付款確認、法遵審核。
- 長文文件閱讀。
- 嚴肅企業後台作為主視覺。

## 資訊密度

中。活動頁首屏要聚焦一個主要任務；任務列表與排行榜可提高密度，但每個任務需有明確進度、獎勵與 CTA。

## 實作優先順序

1. 建立 MissionCard、ProgressBar、RewardCard、Leaderboard。
2. 建立完成任務後的成功回饋與領獎流程。
3. 建立活動規則、剩餘時間、資格限制與錯誤狀態。
4. 補後台設定：任務、獎勵、名單、數據。

## 風格邊界

- 可以鮮明與有動效，但不得影響可讀性或造成誤導。
- 抽獎、獎勵、積分規則必須透明。
- 不用賭博感過強的視覺或誘導性文案。
