# Components：遊戲化活動互動介面

## Button

- Primary：開始任務、領取獎勵、立即參加。
- Secondary：查看規則、分享活動、查看紀錄。
- Ghost：任務詳情、排行榜切換。
- Danger：取消活動、移除任務、撤銷獎勵（後台）。

規則：

- 活動前台 CTA 可高對比。
- 未符合資格時 disabled 並顯示原因。
- 領獎按鈕需避免重複提交。

## Event Hero

結構：

1. 活動標題。
2. 活動副標。
3. 剩餘時間。
4. 主要 CTA。
5. 活動期間與資格摘要。
6. 視覺背景。

規則：

- 倒數需可讀，不只視覺裝飾。
- 資格限制不可藏在最底部。

## Mission Card

結構：

1. 任務名稱。
2. 任務說明。
3. 進度。
4. 獎勵。
5. 狀態 badge。
6. CTA。

狀態：

- locked：未解鎖。
- available：可開始。
- in-progress：進行中。
- completed：已完成待領取。
- claimed：已領取。
- expired：已過期。

## Progress Bar / Ring

- 顯示百分比與完成數。
- 可顯示下一個里程碑。
- 不只用顏色表示進度。
- 動畫不造成文字跳動。

## Reward Card

結構：

- 獎勵名稱。
- 獎勵數量或價值。
- 取得條件。
- 有效期限。
- 領取狀態。

規則：

- 虛擬點數、券、實物需清楚區分。
- 過期或已領取狀態要明確。

## Leaderboard

欄位：

- 排名。
- 使用者。
- 分數。
- 變化。
- 獎勵區間。

規則：

- 我的排名固定顯示或提供快速定位。
- 分數相同的排名規則需說明。
- 不暴露不該公開的個資。

## Countdown

- 顯示天/時/分/秒。
- 結束後切換為活動結束狀態。
- 不使用造成 reflow 的變動寬度字體。

## Claim Reward Modal

必備：

1. 獎勵摘要。
2. 使用限制。
3. 有效期限。
4. 確認 CTA。
5. 成功/失敗狀態。

## Rule Panel

- 活動期間。
- 參加資格。
- 任務計算方式。
- 獎勵發放方式。
- 注意事項。

規則文字不能太小，重要限制需 highlight。

## Activity Log

- 任務完成時間。
- 積分變動。
- 獎勵領取紀錄。
- 失敗原因。

## Admin Mission Editor

欄位：

- 任務名稱。
- 任務條件。
- 獎勵設定。
- 開始/結束時間。
- 可見對象。
- 發布狀態。

高風險變更需確認影響範圍。

## Empty State

- 尚無任務：顯示下一波活動或返回首頁。
- 無排名：說明完成任務後上榜。
- 無獎勵：引導先完成任務。
